# StudyFlow OS 答辩演示脚本

## 演示环境准备

- [ ] HarmonyOS NEXT 模拟器或真机启动
- [ ] DevEco Studio 项目已同步
- [ ] Mock 模式确认开启（`Constants.USE_MOCK = true`）
- [ ] 投屏设备已连接（HDMI/无线投屏）
- [ ] 10分钟演示时间规划

---

## 演示流程（总时长 8-10 分钟）

### 开场 (30秒)

> "各位老师好，我做的项目是**基于 HarmonyOS NEXT 与 DeepSeek V4 的大学生学习任务操作系统**——StudyFlow OS。
>
> 这不是一个普通的AI聊天App，而是一个围绕'课程理解 → 知识结构化 → 任务拆解 → 复习检测 → 反馈再规划'的完整学习闭环系统。"

### 第一阶段：学习驾驶舱 HomePage (1分钟)

**展示内容：**
1. 首页概览 — 4 个统计卡片（今日任务/即将截止/薄弱点/待复习）
2. 向下滑动展示各个区域
3. 重点指出 Mock 模式横幅："当前使用示例数据运行，无需网络"

**台词：**
> "首页是学习驾驶舱，学生打开就能看到今天必须完成什么、什么快截止了、哪些知识点薄弱。这不是简单的待办列表，而是经过 AI 分析后生成的结构化任务。"

### 第二阶段：课程资料输入 CourseInputPage (1分钟)

**操作步骤：**
1. 点击"➕ 新建课程资料"
2. 点击"📥 使用示例资料"
3. 观察自动填充的课程内容（概率论与数理统计）

**台词：**
> "系统支持 6 种类型的文本资料输入。第一版我们不做复杂的 PDF 解析，而是聚焦在文本结构化上。点击'使用示例资料'可以快速加载概率论与数理统计的课程内容。"

### 第三阶段：AI 课程解析 CourseAnalysisPage (2分钟)

**操作步骤：**
1. 点击"🤖 AI 解析课程"
2. 进入 CourseAnalysisPage
3. 从上到下滑动展示：
   - 课程摘要
   - 章节结构（4个章节）
   - 核心知识点（8个）
   - 考试重点（标注分值预测）
   - 作业任务（标注截止时间）
   - 学习风险提醒（标注风险等级）

**台词：**
> "这是系统的核心能力——AI 课程理解。DeepSeek 分析课程资料后，自动提取出结构化的课程画像。大家可以看到，不仅有知识点列表，还有考试重点预测、作业任务提取和学习风险提醒。这些都是 AI 自动生成的，不是人工整理的。"

### 第四阶段：知识点结构 KnowledgeMapPage (1分钟)

**操作步骤：**
1. 点击"🧠 查看知识点结构"
2. 展示按章节分组的知识点卡片
3. 点击章节筛选 Tab 切换

**台词：**
> "知识点结构页按章节分组展示，标注了每个知识点的重要程度和难度。第一版我们没有做复杂的知识图谱，而是用分组卡片替代，既稳定又清晰。"

### 第五阶段：复习卡片 ReviewCardPage (2分钟) — **核心演示**

**操作步骤：**
1. 返回 CourseAnalysisPage
2. 点击"🃏 生成复习卡片"
3. **第1张卡片**：先展示知识点解释和重点提醒
   - 点击"查看答案与解析"
   - 选择"🤔 还模糊"
   - 观察反馈结果和二次复习建议
4. **第2张卡片**：展示另一道题
   - 点击"查看答案与解析"
   - 选择"❌ 做错了"
   - 观察紧急复习任务生成

**台词：**
> "这是学习闭环的关键节点——复习卡片。每张卡片包含知识点解释、重点提醒、常见错误和一道自测题。学生做题后点击反馈按钮，系统不仅记录结果，还会立即生成个性化的二次复习建议。
>
> 比如我选择'做错了'，系统建议今天内重新学习，并自动生成紧急复习任务。这就是'反馈闭环'。"

### 第六阶段：复习计划 PlanPage (1分钟) — **闭环展示**

**操作步骤：**
1. 点击"📋 查看复习计划"
2. 展示统计卡片（任务总数和完成情况）
3. 展示最近反馈
4. 切换 Tab：薄弱点 → 看到刚生成的强化任务
5. 展示二次复习建议

**台词：**
> "PlanPage 展示了学习闭环中的'再规划'结果。刚才反馈'做错了'的知识点，这里立刻出现了强化任务。系统还会根据所有反馈生成综合建议——比如每天投入2小时复习、优先解决方差计算问题等。"

### 第七阶段：AI Agent 架构说明 (1分钟)

**台词（不操作页面）：**
> "最后介绍一下技术架构。系统的 AI 能力分成四层：
>
> - **最底层**是 DeepSeekClient，负责 HTTP 通信
> - **第二层**是 PromptBuilder，所有 Prompt 要求严格 JSON 输出
> - **第三层**是三个 AI Agent：CourseParserAgent（课程解析）、ReviewPlannerAgent（复习规划）、FeedbackAgent（反馈分析）
> - **页面层**不允许直接调用 DeepSeek API，必须通过 Agent 服务层
>
> 当 API Key 缺失或网络故障时，系统自动降级到 MockData，保证答辩现场稳定演示。"

### 收尾 (30秒)

> "总结一下，StudyFlow OS 实现了'输入 → 理解 → 结构化 → 检测 → 反馈 → 再规划'的完整学习闭环。系统基于 HarmonyOS NEXT 原生开发，使用 ArkTS + ArkUI，AI 能力由 DeepSeek V4 提供。
>
> 谢谢各位老师，请批评指正。"

## QA 预案

### Q1: 为什么选择 HarmonyOS NEXT？
> HarmonyOS NEXT 是国产全栈自研操作系统，ArkTS + ArkUI 声明式开发效率高。作为本科毕业设计，选择国产平台具有前瞻性和社会价值。

### Q2: 和普通 AI 聊天 App 有什么区别？
> 普通 AI 聊天是对话式的、无状态的。StudyFlow OS 是一个有状态的任务操作系统——它记住你的学习进度、分析你的薄弱点、动态调整复习计划，形成了可追踪的闭环。

### Q3: 知识点图谱为什么不做成复杂的图？
> 第一版聚焦核心闭环的可用性和稳定性。复杂的图谱算法和 Canvas 渲染在原型阶段容易出 bug，分组卡片方案同样能展示知识结构，而且答辩时更稳定。

### Q4: Mock 数据是手动写的还是 AI 生成的？
> Mock 数据是参照真实教材内容构建的结构化数据，用于验证系统逻辑和兜底答辩。在真实使用场景下，这些数据由 DeepSeek API 实时生成。

### Q5: 如果网断了怎么办？
> Mock 模式确保系统在任何情况下都能完整演示。所有核心数据都有本地兜底方案。这是系统的设计原则之一——高可用性。

## 故障预案

| 故障 | 应对方案 |
|------|----------|
| 模拟器卡死 | 重启模拟器，快速回到演示流程 |
| 页面跳转失败 | 手动输入路由路径，或从首页重新开始 |
| 投屏中断 | 改用模拟器窗口演示 |
| 时间不足 | 缩减第二阶段（课程输入），重点保留第五阶段（复习卡片闭环） |
