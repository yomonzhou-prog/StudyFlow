# StudyFlow OS 测试说明

## 测试环境

- **操作系统**: HarmonyOS NEXT
- **测试方式**: Mock 模式手动测试（不依赖 DeepSeek API）
- **测试数据**: 内置三套示例课程（概率论与数理统计、数据结构、大学英语）

## 测试用例

### TC-01: 首页加载与展示

| 项目 | 内容 |
|------|------|
| 测试目标 | 验证 HomePage 正确加载并展示所有区域 |
| 前置条件 | 首次启动应用 |
| 操作步骤 | 1. 启动应用<br>2. 观察首页 |
| 预期结果 | 1. 显示"StudyFlow OS 学习任务操作系统"标题<br>2. 显示 Mock 模式横幅<br>3. 显示4个统计卡片（今日任务/即将截止/薄弱点/待复习）<br>4. 显示"📋 今日必须完成"区域<br>5. 显示"⏰ 三天内截止"区域<br>6. 显示"🎯 薄弱知识点"区域<br>7. 显示"📚 今日复习卡片"区域<br>8. 显示"➕ 新建课程资料"按钮<br>9. 显示"📋 查看复习计划"按钮 |

### TC-02: 课程资料输入 - 使用示例

| 项目 | 内容 |
|------|------|
| 测试目标 | 验证示例资料加载和表单功能 |
| 前置条件 | 在 HomePage |
| 操作步骤 | 1. 点击"新建课程资料"<br>2. 进入 CourseInputPage<br>3. 点击"📥 使用示例资料"<br>4. 观察表单填充 |
| 预期结果 | 1. 课程名称自动填充"概率论与数理统计"<br>2. 资料标题自动填充"第四章：随机变量的数字特征"<br>3. 类型自动选中"教材"<br>4. 文本框填充课程内容<br>5. 显示 Toast "已加载示例课程资料" |

### TC-03: 课程资料输入 - 空值验证

| 项目 | 内容 |
|------|------|
| 测试目标 | 验证必填字段校验 |
| 前置条件 | 在 CourseInputPage，所有字段为空 |
| 操作步骤 | 1. 保持所有字段为空<br>2. 点击"AI 解析课程" |
| 预期结果 | 1. 显示"请输入课程名称"错误提示<br>2. 显示"请输入资料标题"错误提示<br>3. 显示"请输入课程资料内容"错误提示 |

### TC-04: AI 解析跳转

| 项目 | 内容 |
|------|------|
| 测试目标 | 验证从输入到解析的完整跳转 |
| 前置条件 | CourseInputPage 已填写完整 |
| 操作步骤 | 1. 点击"AI 解析课程"<br>2. 观察跳转 |
| 预期结果 | 1. Toast "课程资料已保存，准备AI解析..."<br>2. 跳转到 CourseAnalysisPage<br>3. 页面加载课程画像 |

### TC-05: 课程解析页展示

| 项目 | 内容 |
|------|------|
| 测试目标 | 验证 CourseAnalysisPage 各区域展示 |
| 前置条件 | 已完成解析跳转 |
| 操作步骤 | 1. 查看页面各区域 |
| 预期结果 | 1. Mock 模式横幅<br>2. 课程名称"概率论与数理统计"<br>3. 课程摘要（100-200字）<br>4. 章节结构（4个章节）<br>5. 核心知识点（8个）<br>6. 考试重点（4个）<br>7. 作业任务（2个）<br>8. 学习风险提醒（3个）<br>9. "🃏 生成复习卡片"按钮<br>10. "🧠 查看知识点结构"按钮 |

### TC-06: 知识点结构页

| 项目 | 内容 |
|------|------|
| 测试目标 | 验证 KnowledgeMapPage 按章节分组展示 |
| 前置条件 | 从 CourseAnalysisPage 进入 |
| 操作步骤 | 1. 点击"查看知识点结构"<br>2. 点击不同章节筛选 |
| 预期结果 | 1. 显示课程名称和知识结构标题<br>2. 显示章节筛选栏<br>3. 默认"全部"显示8个知识点<br>4. 点击"4.1"筛选只显示该章节知识点<br>5. 知识卡片显示标题、解释、重要程度、难度 |

### TC-07: 复习卡片 - 完整流程

| 项目 | 内容 |
|------|------|
| 测试目标 | 验证复习卡片查看、答题、反馈完整流程 |
| 前置条件 | 从 CourseAnalysisPage 点击"生成复习卡片" |
| 操作步骤 | 1. 查看第1张卡片<br>2. 阅读知识点解释<br>3. 点击"查看答案与解析"<br>4. 选择一个反馈（已掌握/还模糊/做错了）<br>5. 观察反馈结果<br>6. 点击"下一张卡片" |
| 预期结果 | 1. 显示进度"第1/4张"<br>2. 卡片包含知识点标题、解释、重点提醒、常见错误<br>3. 自测题显示4个选项<br>4. 点击后显示正确答案和解析<br>5. 反馈按钮可见<br>6. 反馈后显示状态和二次复习建议<br>7. 显示"查看复习计划"按钮 |

### TC-08: 反馈闭环验证

| 项目 | 内容 |
|------|------|
| 测试目标 | 验证反馈保存和学习计划更新 |
| 前置条件 | 在 ReviewCardPage 完成至少一次反馈 |
| 操作步骤 | 1. 点击反馈按钮（如"做错了"）<br>2. 点击"查看复习计划"<br>3. 观察 PlanPage |
| 预期结果 | 1. 显示"❌ 做错了"反馈结果<br>2. 生成二次复习建议文本<br>3. PlanPage 显示新的薄弱点任务<br>4. 最近反馈区域包含刚提交的反馈 |

### TC-09: 复习计划页

| 项目 | 内容 |
|------|------|
| 测试目标 | 验证 PlanPage 完整展示 |
| 前置条件 | 从首页或复习卡片页进入 |
| 操作步骤 | 1. 查看统计卡片<br>2. 切换 Tab（今日/明日/薄弱点/临考冲刺）<br>3. 查看二次复习建议<br>4. 切换任务完成状态 |
| 预期结果 | 1. 显示任务统计（总/已完成/进行中/待完成）<br>2. 显示最近反馈<br>3. Tab 切换显示不同任务列表<br>4. 薄弱点 Tab 显示强化任务<br>5. 二次复习建议有文本内容 |

### TC-10: 完整闭环流程

| 项目 | 内容 |
|------|------|
| 测试目标 | 验证从输入到二次规划的完整闭环 |
| 前置条件 | 启动应用 |
| 操作步骤 | 1. HomePage → 新建课程资料<br>2. 加载示例资料 → AI解析<br>3. 查看课程解析结果<br>4. 查看知识点结构<br>5. 生成复习卡片<br>6. 完成4张卡片的反馈（混合选择）<br>7. 查看复习计划<br>8. 返回驾驶舱 |
| 预期结果 | 1. 全流程无报错<br>2. 每个页面正确展示数据<br>3. 反馈后计划页有新任务<br>4. 导航按钮正确跳转<br>5. Mock 模式横幅始终可见 |
